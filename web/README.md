# Консоль обзвона — веб-дашборд лидов

Веб-интерфейс поверх `commercial_realty.xlsx` для удобного обзвона: поиск, фильтры,
карточки с фото/телефоном/адресом/гео + действия (сохранить объявление, открыть в Excel,
обновить базу). Дизайн сделан скилом **impeccable**.

## Как открыть

**Двойной клик по `Дашборд.command`** в корне проекта. Он обновит данные, поднимет
локальный сервер с API и откроет дашборд в браузере. **Окно терминала не закрывайте, пока
пользуетесь** — оно держит сервер (остановить: Ctrl+C или закрыть окно).

Вручную: `./bin/python web/server.py` (затем открыть напечатанный адрес
`http://localhost:8765/index.html`).

> **Важно:** работать через сервер (http://localhost), а не открывать `index.html` как файл.
> Из `file://` Safari блокирует открытие ссылок и кнопки «Сохранить/Обновить/Excel» не работают
> (им нужен бэкенд-сервер).

## Что умеет

**Просмотр и фильтрация**
- Поиск по адресу, типу, телефону, источнику, описанию.
- Фильтры: сделка (продажа/аренда), город, тип, источник, «с телефоном», «с фото».
- Сортировка: свежие, цена ↑/↓, площадь ↓.
- Клик по зелёной кнопке с номером — копирует телефон в буфер.

**Действия на карточке**
- **Сохранить** — скачивает полную веб-версию объявления офлайн: до-фетчит страницу
  (полный текст + все фото), кладёт в `web/saved/{хэш}/` и собирает автономную
  `index.html`. Кнопка превращается в «Сохранено» — ссылку на эту копию.
  Работает для kufar (текст+фото из JSON), megapolis и realt (текст из HTML, фото из og:image).
- **Excel** — открывает `commercial_realty.xlsx` и выделяет строку этого объекта
  (через AppleScript). Первый раз macOS может спросить разрешение на управление Excel — разрешите.

**Обновление базы**
- Кнопка **«Обновить базу»** (вверху) запускает `collect_realty.py` (инкрементальный сбор
  свежих объявлений со всех источников) прямо из браузера, показывает живой лог в окне,
  по завершении перевыгружает данные и перезагружает страницу. Сбор идёт в фоне — окно лога
  можно закрыть. Это полноценный скрейп: может занять много минут и упереться в антибот.

## Файлы

| Файл | Что это |
|---|---|
| `index.html` · `styles.css` · `app.js` | интерфейс (дизайн-система + логика) |
| `server.py` | локальный сервер + API (`/api/save`, `/api/reveal`, `/api/update`) |
| `fetch_ad.py` | до-фетч полной веб-версии объявления (текст + фото) |
| `export_data.py` | выгрузка xlsx → `data.js` |
| `data.js` | сгенерированные данные (не редактировать; в .gitignore) |
| `saved/` | сохранённые офлайн-копии объявлений (в .gitignore) |

## Заметки

- **Фото и сбор требуют доступа к .by-сайтам** — включите Psiphon (как для парсера).
  Если объявление за антиботом (kufar-телефон), сохранится то, что доступно без капчи
  (текст + фото есть, телефон берётся из строки таблицы).
- realt.by фото для карточек-превью не отдаёт, но при **Сохранении** фото realt
  достаются из og:image — в офлайн-копии они есть.
- Данные локальны, никуда не отправляются.
